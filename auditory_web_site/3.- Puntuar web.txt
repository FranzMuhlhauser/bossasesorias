# CATEGORÍAS DE EVALUACIÓN (0-10)
1. SEO Técnico: Indexabilidad, metadatos, estructura, Schema, rastreo.
2. AEO: Formato pregunta-respuesta, FAQs, markup directo, probabilidad de Featured Snippet.
3. GEO: Claridad semántica, densidad de conocimiento, autoridad temática, citabilidad por LLMs.
4. Performance & Build: Minificación, tree-shaking, code splitting, eliminación de dead code, caché.
5. Core Web Vitals: LCP <2.5s, CLS <0.1, INP <200ms, priorización de carga crítica.
6. Optimización de Medios: WebP/AVIF, compresión, srcset, lazy loading, dimensiones explícitas.
7. Responsive & UX Móvil: Viewport, breakpoints lógicos, tap targets >48px, sin overflow, fluidez.
8. Experiencia de Usuario (UX): Jerarquía visual, navegación intuitiva, feedback de estados, fricción cero.
9. Accesibilidad (WCAG 2.2 AA): Contraste ≥4.5:1, focus visible, ARIA correcto, navegación por teclado.
10. Seguridad & Pre-Producción: HTTPS, headers (CSP, X-Frame, Referrer), sanitización, sin secrets/console.log.

# METODOLOGÍA DE PUNTUACIÓN (obligatoria)
- 9-10: Implementación óptima. Lista para producción sin ajustes.
- 7-8: Correcta, pero requiere optimizaciones menores pre-despliegue.
- 5-6: Deficiente en varios puntos. Riesgo de retrabajo o mala experiencia.
- 3-4: Problemas estructurales que afectan SEO, rendimiento o seguridad.
- 0-2: Crítico. Bloquea entrega o expone vulnerabilidades/deuda técnica masiva.
